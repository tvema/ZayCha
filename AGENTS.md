# Правила проекта и зафиксированные решения

Этот файл содержит ключевые архитектурные решения и инструкции, которые нельзя нарушать или изменять без явного указания пользователя.

## Замороженные и отлаженные модули (Не изменять без запроса!)

### 1. Просмотрщик PDF (`components/DocumentViewer.tsx`)
- **Статус:** Отлажен и работает.
- **Особенности:** Worker для `pdfjs-dist` берется строго из локального файла `/pdf.worker.min.mjs` внутри `public`, чтобы избежать проблем с доступностью внешних CDN и версионированием. Сама библиотека должна импортироваться из legacy сборки: `await import('pdfjs-dist/legacy/build/pdf.mjs')`, иначе Next.js выдает ошибку о неопределенном DOMMatrix.
- **Инструкция:** НЕ менять способ загрузки воркера. НЕ пытаться скачивать его с `cdnjs` или `unpkg`. НЕ менять импорт основной библиотеки на обычный `pdfjs-dist`.

### 2. Иконки приложения (PWA Manifest)
- **Статус:** Сконфигурировано.
- **Особенности:** Иконки пользователя были вручную загружены в папку `public` (`icon-16x16.png`, `icon-32x32.png` и т.д.).
- **Инструкция:** НЕ удалять эти иконки. НЕ генерировать новые иконки через сторонние скрипты, если только пользователь прямо не попросит обновить иконки. `manifest.json` и `app/layout.tsx` уже корректно ссылаются на них.

### 3. Модальные окна, AnimatePresence и backdrop-blur
- **Статус:** Отлажено (решена проблема "зависшего заблюренного экрана").
- **Особенности:** Если модалка с `backdrop-blur` закрывается, фильтр и слой с кликами могут "зависать", если не анимировать демонтирование внешней обертки правильно.
- **Инструкция:**
  1. Внешняя обертка модалки (затемнение фона) **обязательно** должна быть `<motion.div>`, а не обычным `<div>`.
  2. У внешней обертки **обязательно** должен быть уникальный `key` (например `key="profile-modal"` или `key={"group-" + id}`).
  3. **Критически важно:** состояние `exit` у внешней обертки модалки должно сбрасывать фильтр и клики: `exit={{ opacity: 0, backdropFilter: 'blur(0px)', pointerEvents: 'none' }}`.
  4. Состояние `exit` внутренней карточки также должно отключать клики `pointerEvents: 'none'`.
  5. НЕ ИЗМЕНЯТЬ эту структуру, если нет прямого запроса пользователя, иначе вернется баг с непропадающим блюром после закрытия окон.
